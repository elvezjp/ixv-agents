# PO Constitution Check

プロジェクトの憲章（CONSTITUTION.md）を確認し、存在意義が記入済みか判定する。

## When to Use

- プロジェクト開始時
- ユーザーが「憲章確認」「目的確認」と言った時
- ワークフロー フェーズ1（原則決定フェーズ）の開始時

## Instructions

### Step 1: CONSTITUTION.mdを読み取り

`CONSTITUTION.md` を読み取り、内容を確認する。

### Step 2: 存在意義セクションを確認

「## 1. 存在意義（Purpose）」セクションの内容を確認する。

**未記入の判定基準**:
以下のいずれかに該当する場合は「未記入」と判定：
- 「このプロジェクトが存在する理由を明記する」というテンプレート文言のまま
- セクション内容が空
- プレースホルダーのまま（例: `{purpose}`、`TODO`）

### Step 3: 結果を報告

**記入済みの場合**:
```
憲章確認完了: 存在意義が記入されています。

目的: {記入されている内容の要約}

→ 次のステップ: フェーズ2 企画・要件定義へ進んでください。
```

**未記入の場合**:
```
憲章確認完了: 存在意義が未記入です。

現在の内容: {現在の内容}

→ 次のステップ:
  1. ユーザーにプロジェクトの目的をヒアリングしてください
  2. po-request-yaml スキル（task_type: constitution_update）で憲章更新タスクを発行してください
```

## Examples

### Example 1: 記入済みの場合

**CONSTITUTION.md の内容**:
```markdown
## 1. 存在意義（Purpose）
- AIエージェントによる効率的なソフトウェア開発プロセスを実現する
```

**報告**:
```
憲章確認完了: 存在意義が記入されています。

目的: AIエージェントによる効率的なソフトウェア開発プロセスを実現する

→ 次のステップ: フェーズ2 企画・要件定義へ進んでください。
```

### Example 2: 未記入の場合

**CONSTITUTION.md の内容**:
```markdown
## 1. 存在意義（Purpose）
- このプロジェクトが存在する理由を明記する
```

**報告**:
```
憲章確認完了: 存在意義が未記入です。

現在の内容: テンプレートのまま

→ 次のステップ:
  1. ユーザーにプロジェクトの目的をヒアリングしてください
  2. po-request-yaml スキル（task_type: constitution_update）で憲章更新タスクを発行してください
```

## Notes

- このスキルはPOロールのみが使用する
- 憲章の変更はHumanの最終承認が必要
- 記入済みの場合はフェーズ2へ、未記入の場合はタスク発行へ進む
